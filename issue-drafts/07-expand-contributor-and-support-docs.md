# Expand contributor docs, triage guidance, and support paths

Labels: `documentation`, `help wanted`, `good first issue`

## Summary

The repo now has better top-level docs, but contributor experience can still be improved with clearer triage guidance, issue labeling guidance, and support-path documentation.

## Scope

- Add a maintainer-facing section on how to triage incoming issues and PRs
- Document which topics belong in GitHub issues vs Discord vs support email
- Add label usage guidance for examples, docs, API, MCP, and CI work
- Keep the guidance lightweight and specific to this repo

## Acceptance criteria

- Contributors can quickly tell where to ask for help
- Maintainers have a lightweight issue-triage guide
- Label usage is documented in-repo
- The new docs do not duplicate large chunks of the product docs

## Notes

Focus on repo operations and contributor clarity, not broad company policy.
